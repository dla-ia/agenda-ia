# Memoria del proyecto AgendaIA

- El proyecto se desarrolla en Next.js 14 + Tailwind + Supabase, con deploy en Vercel y control de versiones en GitHub.
- El flujo de trabajo es: desarrollo local → push a GitHub → deploy automático en Vercel.
- El MVP incluye: panel web, agenda, pacientes, conversaciones, configuración.
- Las integraciones externas (Google Calendar, WhatsApp, Claude API) se implementarán en pasos separados.
- Se prioriza claridad, modularidad y documentación en cada avance.
- Se recomienda siempre tildar tareas y actualizar contexto al cerrar sesión.
- Problemas comunes: errores de deploy por configuración de framework, variables de entorno faltantes, permisos de GitHub.
